## [7.1.1] — 2026-05-03

### 🐛 Bugfix critici

**LiveMarketService — Race conditions risolte**
- **`get_live_market_service()` singleton race**: il pattern
  `if _singleton is None: _singleton = LiveMarketService()` poteva creare
  istanze multiple sotto contesa concorrente. Sostituito con
  double-checked locking esplicito (`_singleton_lock`). Verifica empirica:
  precedentemente con 50 thread paralleli si creavano 50 istanze separate;
  ora ne viene creata UNA sola.
- **`get_kpi_snapshot()` parallel HTTP fetch**: il lock veniva rilasciato
  prima di chiamare `_fetch_and_cache()`, quindi N thread che trovavano
  cache scaduta facevano N download HTTP paralleli. Sostituito con pattern
  single-flight: una `Condition` e un flag `_refresh_in_progress` fanno sì
  che solo il primo thread esegua il fetch, gli altri attendano e leggano
  la cache aggiornata. Verifica: 10 thread → 1 sola chiamata HTTP.
- **`delta_pct` con override manuale**: il delta percentuale mostrato in
  dashboard veniva calcolato come `(override_value - prev_close) / prev_close`,
  producendo numeri fuorvianti (es. con override 999 e prezzo API 100,
  delta sembrava +900%). Ora `delta_pct` riflette sempre il movimento del
  PREZZO API (`(last_close - prev_close) / prev_close`), indipendentemente
  dall'override applicato al valore mostrato.

**EToroParser — Header detection bug**
- `_reposition_header()` non riconosceva le colonne `"Unnamed: 0"`,
  `"Unnamed: 1"` auto-assegnate da pandas quando l'XLSX ha righe
  decorative in cima all'header reale. Il check
  `all(isinstance(c, str) and c.strip() ...)` passava per tali colonne
  (sono stringhe non vuote!) e il metodo usciva senza riposizionare.
  Risultato: tutte le colonne normalizzate diventavano NaN. Aggiunto
  `_is_pandas_autogenerated_header()` che riconosce il pattern.
- **`_normalize_direction()` typing**: signature dichiarata `-> str | float`
  ma ritornava `pd.NA` (tipo `NAType`). Mypy `--strict` rifiutava. Ora
  signature corretta `-> str | None`, e nel chiamante `astype("string")`
  converte `None` in `pd.NA` mantenendo semantica pandas.
- **Alias colonne `Action`**: nei report eToro recenti `Action` indica
  "Buy/Sell" (direction), non il nome dello strumento. Spostato l'alias
  `^action$` da `raw_action` a `direction`.

**SanityChecker — Macro rules ignorate**
- Le regole `macro` definite in `config/sanity_rules.yaml`
  (`unemployment_rate`, `inflation_rate`, `gdp_growth`) non venivano mai
  applicate perché `_load_config()` iterava solo su `_DEFAULT_RULES`,
  che non includeva la chiave `"macro"`. Aggiunti default macro nel codice,
  merge ricorsivo per nested dicts, e nuovo metodo `check_macro_data()`
  con inferenza automatica del tipo di indicatore dal series_id FRED
  (UNRATE → unemployment_rate, CPIAUCSL → inflation_rate, ecc.).

### ✨ Nuove feature

**Integrazione API ufficiale eToro**
- Client `EtoroClient` per la Public API eToro (`api-portal.etoro.com`):
  endpoint `GET /trading/info/real/pnl`, `GET /market-data/instruments`,
  `GET /market-data/rates`, `GET /market-data/search`.
- Authentication via 3 header (`x-api-key`, `x-user-key`, `x-request-id`
  UUID univoco per call), retry esponenziale su 429/5xx, mapping HTTP
  errors → exception classificate (`EtoroAuthError`, `EtoroRateLimitError`,
  `EtoroNetworkError`).
- Pydantic models per le response (`EtoroPosition`, `EtoroInstrument`,
  `EtoroPortfolioResponse`) con `extra="ignore"` per resilienza a cambi API.
- Facade unificato `EtoroImporter`: decide automaticamente tra API
  (se `ETORO_API_KEY` + `ETORO_USER_KEY` in `.env`) e parser XLSX (fallback).
- Pagina **P2 Portafoglio eToro** ridisegnata con tre sub-tab: API
  (preferito), XLSX (fallback), Help (istruzioni step-by-step per
  ottenere le keys da eToro Settings → Trade).
- Il parser XLSX non viene rimosso: rimane come fallback offline e per
  utenti senza API access.

**ApiHealthChecker — Ping reali per pagina E0**
- Nuovo modulo `engine/market_data/hardening/api_health_checker.py` che
  pinga effettivamente Yahoo Finance, FRED, Alpha Vantage, Finnhub e
  classifica lo stato in `ONLINE`/`DEGRADED`/`OFFLINE`/`NO_API_KEY`.
- Detect rate-limited responses anche con HTTP 200 (es. Alpha Vantage
  ritorna body con `"Information"` o `"Note"` quando rate-limited).
- Pagina **E0 API Health** ora mostra stato reale invece di hardcoded
  "ℹ️ N/A" per FRED/AV/Finnhub. Bottone "🔄 Aggiorna stato API" forza
  re-ping con cache in `st.session_state`.
