# Stress Testing Guide

Rule 24: stress tests MUST include both historical and forward-looking
synthetic scenarios. The `engine/stress_testing/` module provides:

- **4 historical scenarios**: Global Financial Crisis 2008,
  COVID Crash 2020, Rate Hike 2022, Dot-Com 2000
- **6 forward-looking scenarios** generated by `ScenarioGenerator` based
  on current market context (VIX, yield curve, sentiment)

## Usage

```python
from engine.stress_testing import StressTester, MarketContext

ctx = MarketContext(
    vix=20.0, yield_curve_2y_10y=0.0,
    sentiment_composite=0.1, regime="transition",
)
tester = StressTester()
report = tester.run(equity_series, ctx)

print(f"VaR 95%: {report.var_95:.1%}")
print(f"CVaR 95%: {report.cvar_95:.1%}")
print(f"P(loss): {report.prob_negative:.0%}")
for outcome in sorted(report.outcomes, key=lambda o: o.max_loss_pct):
    print(f"  {outcome.scenario.name}: max DD = {outcome.max_loss_pct:.1%}")
```

## Forward-Looking Generation

The `ScenarioGenerator` builds 6 synthetic scenarios calibrated on the
current market regime, not on a fixed historical period. This addresses
the criticism that pure-historical stress testing fights the last war.

| Scenario | Equity Shock | Bond Shock | VIX |
|----------|-------------|-----------|-----|
| Inflation Surprise | -15% | -8% | 35 |
| Geopolitical Spike | -20% | +3% | 45 |
| Liquidity Crunch | -25% | -12% | 55 |
| Tech Selloff | -30% | +2% | 38 |
| Bond Crash | -8% | -18% | 30 |
| Stagflation | -18% | -10% | 32 |

Probabilities are calibrated against the current `MarketContext` so they
shift over time.
